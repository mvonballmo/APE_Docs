document.forms

  • Forms sind in einem zusätzlichen Struktur auf dem document verfügbar
  • document.forms ist ein Array, der sämtliche Formulare des Dokuments enthält
  • form.elements ist ein Array, das sämtliche Felder des Formulars enthält
  • In selects Eingaben beinhaltet das Array options die Auswahloptionen

forms Beispiel

          
            

              
              
              
            

          
        
          
            document.forms[0].elements[0].id == "firstName"
            document.vehicles.firstName.id == "firstName"
            document.forms[0].elements[2].options[0].id == "one"
            document.vehicles.numCars.options.one.id == "one"

Jest: skip()

  • Es kann vorkommen, dass ein Test nicht erfolgreich ausgeführt werden kann
  • In jest kann man mit skip() ein test() oder ein describe() überspringen
          
            describe("Forms", () => {
              test.skip("some test that doesn't work yet", () => {
              })
            });

Jest: coverage

  • Bei Tests soll man fragen: wie gut ist die Abdeckung?
  • Man muss nicht unbedingt 100% anstreben
  • Im Terminal jest --coverage ausführen, um die Abdeckung anzuzeigen
  • In WebStorm geht auch "Run ... with Coverage"
    • Die Resultaten werden in einem Tool-Fenster eingeblendet
    • Code-Zeilen werden grün/rot (abgedeckt/nicht abgedeckt) markiert

Eingaben: Einführung

  • Ein modernes Browser bietet eine sehr ausgereifte Komponentenbibliothek an
  • Zudem ist eine ausgereifte Validierung vorhanden
  • Vieles kann im HTML direkt spezifiziert werden
  • Das API in JavaScript deckt noch mehr ab (Ereignisse, Methoden, Eigenschaften)

Allgemeine Eigenschaften

name Der Name der Eingabe (wird mit dem Formular am Server gesendet)
value Initialer oder aktueller Wert
required Gibt an, ob die Eingabe zwingend ist
readonly Gibt an, ob die Eingabe geändert werden kann
disabled Gibt an, ob die Eingabe nicht manipuliert werden kann
autofocus Gibt an, ob die Eingabe beim Laden der Seite den Tastatur-fokus bekommen soll

Text Eigenschaften I

minlength Minimale Länge des Werts einer Text-Eingabe
maxlength Maximale Länge des Werts einer Text-Eingabe
placeholder Der Text, der in einer leeren Eingabe angezeigt werden soll
pattern Gibt an, welcher Reguläre Ausdruck den Wert entsprechen muss

Text Eigenschaften II

autocomplete Gibt an, ob und was für eine Vorschlagsliste angezeigt werden soll
list Weist auf eine Liste von möglichen Werten zu
spellcheck Gibt an, ob die Rechtschreibung geprüft werden soll
inputmode Ein Hinweis welche Tastatur angezeigt werden soll

Diskrete Eigenschaften

min Minimaler Wert (Zahl, Datum, Monat, usw.)
max Maximaler Wert (Zahl, Datum, Monat, usw.)
step Gibt an, welche Abstufung für Werte benutzt werden soll (ist spezifisch pro Eingabetyp)

Validierungen

  • Wird beim "submit" automatisch ausgelöst
  • Kann auf einer Eingabe oder ein Formular mit checkValidity() oder reportValidity() ausgelöst werden
  • Mit setCustomValidity() kann die Validierungsnachricht angepasst werden

Validierung: Beispiel

          
            

              
              
              
              
            

          
        
          
            input:invalid {
              outline: 1px dotted red;
              background-color: mistyrose;
            }

Ereignisse

input Wird bei jeder Änderung des Werts ausgelöst
change Wird erst bei einer Änderung des Werts ausgelöst, wenn wenn Änderungen "abgeschlossen" sind
invalid Wird bei einer fehlgeschlagenen Validierung ausgelöst

Ereignis: input

  • Kann benutzt werden, um "live" Änderungen anzuzeigen
#FFFF00

Ereignis: change

  • Kann benutzt werden, um nur explizite Änderungen anzuzeigen
#FFFF00

Ereignis: invalid

  • Mit preventDefault() kann das Anzeigen von der Browser-Default Validierung unterdrückt werden
Valid 
          
            age.addEventListener("invalid", e => {
              ageOutput.value = "Invalid!";
              e.preventDefault();
            });
            age.addEventListener("input", e => {
              ageOutput.value = "Valid";
            });

Validierung Ausführen

          
            const age4 = document.getElementById("age4");
            const submit4 = document.getElementById("submit4");

            age4.addEventListener("input", e => {
              submit4.disabled = !e.target.checkValidity();
            });
